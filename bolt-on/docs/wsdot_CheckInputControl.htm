WSDOT Custom Scripting for Hyperion

wsdot_CheckInputControl Function

Examines a given input control. Returns a value indicating whether or not a control has a value input or selected.
[ActiveDocument.]wsdot_CheckInputControl (Dashboard, Control)

Arguments

Dashboard
Required. A string value containing the name of the Dashboard that contains the control containing the values to use.


Control
Required. A string value containing the name of the ListBox or TextBox control containing the value(s) to use.



Return Value

The wsdot_CheckInputControl function returns the following values:
If wsdot_CheckInputControl returns
Dashboard is not the name of a Dashboard object. false
...and "Dashboard is not the name of a Dashboard object." is written to the console.
Control is not the name of a control in Dashboard. false
...and "Control is not the name of a control in Dashboard." is written to the console.
Control is the name of a control other than a ListBox or TextBox. false
...and "Control is not a ListBox or TextBox control." is written to the console.
Control has no item selected and/or no text entered. false
Control has an item selected or contains text. true


Remarks

The following examples use wsdot_CheckInputControl to determine whether a value is selected in a ListBox: 

//	output whether lstProgram has at least one value selected
Alert(wsdot_CheckInputControl("Dashboard", "lstProgram"));


Requirements

The WSDOT Custom Scripts from the script repository must be loaded into the document.
