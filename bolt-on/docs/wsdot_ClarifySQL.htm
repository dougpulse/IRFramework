WSDOT Custom Scripting for Hyperion

wsdot_ClarifySQL Function

Makes a Hyperion SQL statement easier to read.
[ActiveDocument.]wsdot_ClarifySQL ([SQL])

Arguments

SQL
Required. The SQL statement to process.



Return Value

The wsdot_ClarifySQL function returns the results of a string processed through certain rules.
If the string is a SQL statement produced by Hyperion, the result is SQL that is easier to read.


Remarks

The following examples use wsdot_ClarifySQL to return an easy-to-read version of the last SQL statement that a query sent to the database server: 

Alert(wsdot_ClarifySQL(ActiveDocument.Sections["Q-Expenditures By Work Order"].LastSQLStatement));


Requirements

The WSDOT Custom Scripts from the script repository must be loaded into the document.
