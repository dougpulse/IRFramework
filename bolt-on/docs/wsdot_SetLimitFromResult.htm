WSDOT Custom Scripting for Hyperion

wsdot_SetLimitFromResults Function

Creates a filter for a Query, Results, or Table section based on selected values from a column in a Results or Table object.
[ActiveDocument.]wsdot_SetLimitFromResult (SourceSectionName, SourceColumnName, TargetSectionName, TargetColumnName, TopicName)

Arguments

SourceSectionName
Required. A string value containing the name of the Results or Table section from which to get the filter values.

SourceColumnName
Required. A string value containing the name of the Item (column) containing the values to use to populate the filter. This argument is not used if the section being filtered is a Results or Table section.


TargetSectionName
Required. A string value containing the name of the Query, Results, or Table section for which to create the filter.


TargetColumnName
Required. A string value containing the name of the Item (field, column) to filter on.


TopicName
Required. A string value containing the name of the Topic (table) containing the Item to filter on. This argument is not used if the section being filtered is a Results section.



Return Value

The wsdot_SetLimitFromResults function returns the following values:
If wsdot_SetLimitFromResults returns
SourceSectionName is not the name of a Results or Table object "Source section name not valid."
SourceColumnName is not the name of an Item in SourceSectionName "Source column name not valid."
TargetSectionName is not the name of a Query, Results, or Table section in the ActiveDocument "Target section name not valid."
TargetColumnName is not the name of a column in TargetSectionName "Target column name not valid."
TargetSectionName is the name of a Query section and TopicName is not the name of a Topic in TargetSectionName "Table name not valid."
All input is valid "Success"


Remarks

The following example uses wsdot_SetLimitFromResults to set a filter on a Query section based on values in a Result section: 

//	The Q-Contracts query returns a list of contract numbers.
ActiveDocument.Sections["Q-Contracts"].Process();
wsdot_SetLimitFromResult("R-Contracts", "Contract Number", 
		"Q-Contract Milestones", "Contract Number", "Contract");
ActiveDocument.Sections["Q-Contract Milestones"].Process();
ActiveDocument.Sections["Contract Milestones"].Activate();


Requirements

The WSDOT Custom Scripts from the script repository must be loaded into the document.
